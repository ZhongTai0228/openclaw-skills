🔥 X/推特 AI 圈今日热议（2月23日）

整理了今天 X 上 AI 圈最火的几条讨论，翻译分享给大家：

---

**1. SkillsMP：Claude Code 的"应用商店"来了**
@sentientt_media 发现有人给 Claude Code 做了一个 Skills 市场（SkillsMP），里面有超过 20 万个 Agent Skills，可以教你的 AI 如何写 PPT、审查 PR、部署云端、分析数据等等。不用复杂的提示词，不用从零开始。（7.1万浏览）

**2. Claude Code 创始人的真实工作流**
@garrytan（YC CEO）分享了一个进阶技巧：当你的 Claude.md 超过 200 行时，可以加入"触发器"——比如当 Claude 碰到特定文件名模式时，自动加载对应的文档。这样既省 token 又高效。引用了 Claude Code 的构建者 Boris Cherny 的工作流：每天并行跑 10-15 个 Claude 会话，终端 5 个 + 网页 5-10 个同时写代码。（10万浏览）

**3. Taste Skill：对抗 AI Slop 的前端设计规范**
@vikingmute 推荐了一个叫 "taste-skill" 的项目，专门约束 AI 生成的前端页面质量——从字体、配色、布局、卡片设计到各种 anti-slop 规则和高级动效。在 AI 生成泛滥的时代，"品味"才是最稀缺的。（1.7万浏览）

**4. Vibe Code 做漂亮 UI 的实战经验**
@om_patel5 分享了 Vibe Coding 美观 UI 的方法论：先用 Excalidraw 画线框图再给 AI 提示词（AI 抄作业比想象力强）；截图喜欢的设计让 AI 参考；用 Tailwind + ShadCN 约束设计系统等。（31万浏览）

**5. 全球 81 亿人，只有 0.3% 在为 AI 付费**
@NoahEpstein_ 发文指出：你的时间线让你觉得 AI 已经饱和了，但数据显示恰恰相反——68 亿人从未使用过 AI。这个市场的机会大到令人尴尬。（121万浏览）

**6. 一个 Agent 可能比十个更好用**
@indigox 分享了与 UBC 大学李霄霄教授的对话：在特定可串行处理的任务中，给一个 Agent 赋予多种技能，往往比多 Agent 协作更快更好。
